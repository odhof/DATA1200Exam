GitHub Collaboration

I arbeidet med denne oppgaven har vi møtt på flere utfordringer knyttet tik bruken av GitHub og GitHub Desktop. Det har vært en bratt læringskurve, spesielt når det gjelder å lære seg et nytt verktøy samtidig som vi skulle samarbeide om endringer i de samme filene.

Oppstartsproblemer

  • Utfordringer med manuell installering av Git
  • Problemer med å koble VS Code til GitHub-kontoen
  • Forvirring rundt staging av endringer
  • Usikkerhet rundt når man skulle committe/pushe

Synkroniseringsproblemer

  • Synkroniseringsproblemer mellom lokale filer og repository
  • Push/pull feilet på grunn av usynkroniserte endringer
  • Merge conflicts når flere jobbet i samme fil
  • Utfordringer med å løse konflikter i GitHub-grensensittet

Vi trodde at vi så lenge vi ikke endret samme linje i en fil, ville alt gå greit. Men vi oppdaget at selv når vi jobbet i forskjellige deler av koden, kunnne det oppstå konflikter når linjene med kode var så lange at vi endte opp med å endre samme linje som en annen.

For å håndtere disse utfordringene implementerte vi flere løsninger:

  • Vi opprettet egne kladdeark hvor vi jobbet uavhengig, før vi limte koden inn i den riktig filen
  • Vi fokuserte på god kommunikasjon og hadde hyppoge fysiske møter
  • Vi hjalp hverandre med koden og informerte hverandre når vi var klare til å committe

Vi kunne ha utforsket branches i større grad for å unngå merge conflicts, men ettersom vi hadde hatt såpass mye problemer med Git i begynnelse, var vi skeptiske til å utforskje enda en funksjon som vi ikke var kjent med.

Læringspunkter

I etterkant ser vi at vi burde ha implemneter branches, da man i utgangspunktet aldri bør jobbe direkte i main – noe som blir spesielt tungvint i større prosjekter. Ved å opprette separate branches for hver feature, kunne vi ha unngått flere av problemene som oppstod. Dette er verdifull erfaring som vi tar med oss videre til neste prosjekt

GitHub Desktop skjermbilder

Skjermbildene viser commits fra to ulike gruppemedlemmer. Selv om profilbildene ser nesten identiske ut, er endringene gjort av forskjellige personer i gruppen. Dette kan man i utgangspunktet se i eksempel bilde 1 og 2: commits history, hvor både [Navn1] og [Navn2] har bidratt med ulike endringer gjennom prosjektet. Disse navnene er visket vekk. Ser man godt etter, er det en liten forskjell i farge og mønster.
